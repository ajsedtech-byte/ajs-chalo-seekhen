What to Expect

Here's how this program works and what we expect from you. Read this before Day 1.

🎯

Weeks 7–8Feature Development Sprint (2 Weeks)
▾
🎯 Sprint Goal
This is the most important 2 weeks of Month 2. You will own a complete feature from idea to deployment. You'll write the spec, design the database, build the UI, write tests, and deploy it live.

Your 10-Day Sprint

Days 1–2
Plan & Spec — Write a 1-page document: What problem does this solve? Who is it for? What's the database schema? What are the API endpoints? What does the UI look like? What could go wrong?
Day 3
Design Review — Present your spec to your mentor. Get approval before writing any code. Create wireframes (pen-and-paper is fine). This prevents you from building the wrong thing.
Days 4–5
Build Backend — Create database tables, write Row Level Security policies, build API routes or Server Actions, add input validation with Zod. Test every endpoint manually.
Days 6–7
Build Frontend — Create React components, connect them to your APIs, handle loading/error/empty states, make everything responsive (works on phone + desktop).
Day 8
Test & Integrate — Connect everything together. Test the full user flow. Fix edge cases. Write automated tests. Try to break your own feature.
Day 9
Code Review — Open a pull request. Respond to feedback. Refactor where needed. Have another intern review your code too.
Day 10
Deploy & Demo — Deploy to Vercel staging. Give a 10-minute live demo to the team. Collect feedback. Merge to main branch if approved.
Pro Tip

The spec is the most important part. Many junior developers start coding immediately and then realize halfway through that they built the wrong thing. Spending 2 days planning saves you 5 days of rework.

Sprint Quality Checklist

  • Database schema + RLS policiesTables created, relations defined, users can only access their own data
  • API layer with validationAll inputs validated with Zod. Proper error responses (not just 500 errors).
  • Responsive UI with all statesWorks on mobile & desktop. Shows loading spinners, error messages, and empty states.
  • TypeScript — zero "any"Every variable and function is properly typed. No shortcuts.
  • Automated testsAt least unit tests for core logic and one integration test for the API.
  • Deployed & demo'dLive on a Vercel preview URL. Team has seen it work.

About "Deep Work"

FAQ

Frequently Asked Questions

Real questions from past interns. Read these before Day 1.

What if I don't know React or TypeScript yet?▾
That's completely fine. Week 2 is specifically designed to teach you the full tech stack from the ground up. We only require basic HTML/CSS/JavaScript knowledge. Many successful interns started without React experience. What matters most is your willingness to learn quickly and ask questions.
How much coding will I do vs. learning?▾
Month 1 is roughly 50% learning, 50% coding. Month 2 flips to 20% learning, 80% coding. Month 3 is almost entirely coding and building. We front-load the learning so you can be productive for the majority of the internship. Even the "learning" parts involve writing code — we don't do passive lectures.
What if I get stuck on something?▾
The 30-minute rule: try to solve it yourself for 30 minutes. Google it, read docs, try different approaches. If you're still stuck after 30 minutes, ask your mentor or post in the team chat. Getting stuck is normal. Staying stuck silently for days is not. Nobody will judge you for asking questions — we'll judge you for not asking.
Will I work on real production code?▾
Yes. From Week 1, you'll be working in the same codebase that powers the live product. Your code goes through the same code review process as everyone else's. By Month 2, your features will be deployed to real users. This isn't a sandbox — it's the real thing.
What happens after the 3 months?▾
Three possible outcomes: (1) A full-time job offer if you've performed well and we have an open position. (2) An extended internship or part-time collaboration. (3) A strong reference letter and a portfolio project you can show to any employer. We're transparent about this from Day 1.
What if I fall behind the schedule?▾
The schedule is a guide, not a rigid deadline. Some weeks you'll move faster, some slower. Your weekly 1-on-1 with your mentor is specifically for recalibrating. If you're consistently behind, we'll adjust the scope — not the timeline. What matters is steady progress and honest communication, not hitting every checkbox on Day X.
Do I need my own laptop?▾
Yes, you'll need a laptop that can run Node.js, VS Code, and a modern browser. 8GB RAM minimum, 16GB recommended. Any operating system works (Windows, Mac, or Linux). If hardware is a constraint, let us know early — we may be able to help.
Is this remote or on-site?▾
We'll confirm the format during your offer discussion. Both modes work, but on-site interns typically ramp up faster due to in-person pair programming and whiteboard sessions. Remote interns need to be extra proactive about communication.
